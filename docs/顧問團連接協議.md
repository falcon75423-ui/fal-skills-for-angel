# 顧問團 ↔ 偵察兵 連接協議

> 適用對象：你的 Mac 上**已經裝了**「顧問團 skill」的人。
> 如果還沒裝顧問團，就跳過這份，深度研究還是可以單獨用，只是無法跑「聯合研判」。

---

## 為什麼要 patch

「深度研究 skill」（偵察兵）有一個機制叫 **聯合研判**——
偵察兵負責「研」（出去找情報），顧問團負責「判」（多角度分析），來回交替直到收斂。

這個機制需要**雙邊都認得對方**：
- 偵察兵這邊已經寫了「聯合研判（偵察兵 × 顧問團）」段落 ✅
- 顧問團那邊也要有對應的「聯合研判（顧問團 × 偵察兵）」段落

如果你顧問團裝的版本沒有那段，「聯合研判 [主題]」這個指令會半邊接不上。

---

## 怎麼 patch

### 步驟 1：開啟你顧問團的 SKILL.md

```bash
open -e ~/.claude/skills/顧問團/SKILL.md
# 或用你習慣的編輯器：code / vim / nano 都可以
```

### 步驟 2：搜尋「生態系接口」段落

按 Cmd+F 找關鍵字「生態系接口」或「聯合研判」。

- 如果**有找到**「### 聯合研判（顧問團 × 偵察兵）」段落 → 你的版本已經支援，**不用 patch**，跳到驗證。
- 如果**沒找到** → 繼續步驟 3。

### 步驟 3：把以下段落貼進「生態系接口」段裡

如果你連「## 生態系接口」這個段都沒有，就在 `## 互動指令` 段之前新增：

```markdown
## 生態系接口

### 聯合研判（顧問團 × 偵察兵）

複雜議題的遞迴協作模式——偵察兵負責「研」，顧問團負責「判」，來回交替直到收斂。

**顧問團在聯合研判中的角色**：接收偵察兵帶回的情報，跑完整討論週期，產出兩樣東西：
1. **判斷**：基於現有情報的分析結論
2. **資訊缺口清單**：「我們還需要知道 X 才能做更好的判斷」——這些缺口回傳給偵察兵做下一輪補搜

**啟動方式**：
- 使用者說「聯合研判 [主題]」→ 偵察兵先出去搜，情報送到顧問團
- 使用者帶著資料說「聯合研判 [主題]」→ 顧問團先分析，發現缺口再派偵察兵
- 「聯合研判 [主題] 跑 N 輪」→ 自動模式，中間不停

**節奏**：顧問團跑完完整週期才交棒給偵察兵。後續輪次的輪數由顧問團自行判斷。

**收斂**：當顧問團一輪結束後沒有新的資訊缺口，聯合研判自然結束，產出最終研判報告。
```

### 步驟 4：把以下兩條互動指令補進顧問團的「互動指令」表

找到顧問團 SKILL.md 裡的 `## 互動指令` 表格，加兩行：

```markdown
| 「聯合研判 [主題]」 | 聯合研判 | 啟動偵察兵×顧問團遞迴協作（逐步模式） |
| 「聯合研判 [主題] 跑 N 輪」 | 聯合研判 | 自動模式，跑 N 輪或跑到收斂 |
```

### 步驟 5：存檔，重新載入

```bash
# 在 Claude Code 對話框：
/reload-plugins
```

---

## 驗證

開新對話，輸入：

```
聯合研判 「2026 年自架 LLM 的成本效益」
```

應該看到的反應：
1. **偵察兵**先出場，列出偵察計畫請你確認
2. 你同意後，偵察兵搜一輪，回來繳令
3. **顧問團**接手，五個角色（老謀／算盤／藝術家／記者／定錨）依序分析
4. 顧問團跑完一週期，列出資訊缺口
5. 缺口又回傳給偵察兵補搜……

如果只有偵察兵或只有顧問團出場，沒有切換，就是 patch 沒生效——重看一次步驟 3 是否完整貼進去了。

---

## 為什麼不直接覆蓋你的顧問團 SKILL.md

因為你顧問團裡可能還有自己的客製化（譬如多了角色、調整了風格），
直接覆蓋會把你的客製化吃掉。**只 patch 一段是最安全的做法**。
