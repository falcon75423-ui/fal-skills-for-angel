> 讀取時機：準備出刊時載入。包含每日精選報、聽風、巡兵器的完整輸出格式。

**語言規則**：
- 來源不限語言，品質 > 語種
- 每日精選報：全繁體中文
- 法眼點評：全繁體中文
- 原文標題：保留原文，附中文翻譯
- NotebookLM 摘要/報告指令：指定繁體中文輸出

## 聽風格式

```
🌬️ 聽風 — [日期] ｜ X 上在聊什麼

今日掃了 X 條討論，留 X 個話題上版面。

───────────────────────────────────

💬 #1 ｜ [話題名稱]
熱度：🔥🔥🔥（高/中/低）
核心討論：一到兩句話說清楚在吵什麼
代表發言：
  → @xxx：「...」
  → @yyy：「...」
📎 討論串連結

法眼快評：一句話

───────────────────────────────────

💬 #2 ｜ ...（同樣格式）
```

**與出刊的關係**：聽風結果附在每日精選報尾段，作為獨立區塊。

## 聽風搜尋策略

**主力平台**：X（Twitter）
**未來觀察**：Threads（官方 API 需 App Review + 商家驗證，門檻過高，暫不納入自動化。2026-03-26 決定暫緩）

| 層 | 做什麼 | 工具 |
|----|--------|------|
| **關鍵字掃描** | 搜尋 AI 相關關鍵字在 X 上的當日熱門討論 | Exa / WebSearch |
| **種子帳號** | 追蹤來源名單中 X 帳號的當日發文 | Exa / WebSearch |
| **話題追蹤** | 從熱門討論往下追引用、回覆、QT | WebFetch |

**搜尋關鍵字範例**（動態調整）：
- `AI agent`、`LLM`、`Claude`、`GPT`、`open source AI`
- `MCP`、`context window`、`RAG`、`fine-tuning`
- 當週特定熱點（如新模型發布、重大事件）

**篩選標準**（比巡稿寬鬆，因為社群討論的價值在即時性）：
- 有實質觀點或一手經驗 → ✓
- 引發大量有品質的討論串 → ✓
- 純轉發、純情緒反應、純迷因 → 跳過
- 新聞搬運沒有自己觀點 → 跳過

## 巡兵器格式

```
⚔️ 巡兵器 — [日期] ｜ Skill / 開源工具雷達

本週掃了 GitHub Trending + SkillsMP + Smithery，留 X 個上版面。

───────────────────────────────────

🔥 [Skill 名稱]
⭐ 1.2K（+890 本週）｜ 語言：TypeScript
一句話：用自然語言控制 Figma 設計稿的 MCP server
📎 repo / 平台連結

法眼聊兩句：（不是技術文件摘要，是法眼用自己的話
說這東西為什麼值得注意、能幹嘛、潛力在哪、
有什麼但書。語感像在跟你聊天，不是在唸規格表。）

───────────────────────────────────

✓ [Skill 名稱]
⭐ 340（+210 本週）｜ 語言：Python
...（同樣格式，摘要可以短一些但核心資訊不能少）

───────────────────────────────────

👀 [Skill 名稱]
⭐ 89（+75 本週）｜ 語言：Rust
...（觀望級，一兩句說清楚為什麼先記著就好）
```

**與出刊的關係**：巡兵器結果附在每日精選報中，位於聽風之後，作為獨立區塊。

## 巡兵器搜尋策略

**來源名單**：見 `sources/skills.md`

| 層 | 做什麼 | 工具 |
|----|--------|------|
| **GitHub Trending** | 掃 weekly trending，篩 skill / MCP server / agent tool / CLI plugin 相關 repo | WebSearch / WebFetch |
| **Skill 市集** | SkillsMP、SkillStore、ClaudeMarketplaces 近一週新上架或攀升最快 | Exa / WebSearch |
| **MCP 生態** | Smithery、mcp.so、Glama 新增或熱門 server | Exa / WebSearch |
| **社群信號** | Reddit、HN、X 上被討論的新工具 | Exa / WebSearch |

**評估框架**：採用兵器評估協議快篩模式（見 `references/兵器評估協議.md`），法眼心裡跑一遍 checklist，不需要全部展開在版面上：

```
□ 動量：週增量明顯？還是停滯？
□ 年齡：太年輕要打折
□ README：3 秒內能看懂這東西幹嘛？
□ 目錄：打開 repo 第一眼，結構清楚嗎？
□ CI：有沒有在跑測試？
□ 最後 commit：多久前？
□ 口碑：Twitter/HN 有人提嗎？說什麼？
□ 痛點：這解決的是真問題還是玩具問題？
□ 生態位：同類型最強的是誰？它有什麼不同？
```

**三分制**：
- 🔥 **驚豔** — 數字健康 + 工程認真 + 口碑正面 + 解決真痛點
- ✓ **值得試** — 多數維度及格，有亮點但也有但書
- 👀 **觀望** — 概念有趣但太早期，或某個維度明顯弱

**摘要規則**：
- Skill 具功能性，摘要要比文章推薦更長
- 必須帶到：目的、能做什麼事、有什麼功能、有什麼潛力
- **不用工程師語言**——轉譯標準參考 NotebookLM 報告 prompt 的語感，但保持法眼個性，不是正式報告
- 想像在跟 使用者聊天：「這東西幹嘛的？為什麼你應該知道它？」
- Skill 不存入 NotebookLM（功能性內容非文章，不適合報告化）

## 巡兵器交棒單

當使用者對巡兵器推薦的工具表達興趣（「這個有興趣」「#2 深挖」「幫我開交棒單」等），法眼輸出以下 **code block** 供使用者複製給後續深審流程：

```
## 🔍 兵器審計請求

**Repo**: {repo URL}
**一句話**：{這東西幹嘛的}
**法眼快篩**：
  - 動量：{star 數 + 週增量}
  - 表面：{README/結構/CI 一句話}
  - 口碑：{Twitter/HN 一句話}
  - 法眼說：{品味判斷一句話}
**深審清單**：
  - [ ] 跑巡兵器深審
  - [ ] 架構圖譜分析
  - [ ] 安裝可行性評估
**特別想知道**：{使用者額外提的問題，沒有就刪掉這行}
```

**注意**：交棒單內容來自法眼已做的快篩，不需要額外跑搜尋。直接把版面上的資訊填入格式即可。
