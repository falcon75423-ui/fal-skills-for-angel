## python-docx 地雷

### SAFE / UNSAFE / RISKY 矩陣

| 操作 | python-docx | 風險等級 | 說明 |
|------|------------|---------|------|
| 新建文件 | ��� | SAFE | 完全控制 |
| 讀取文件 | ✅ | SAFE | 唯讀模式 |
| 修改簡單文件 | ⚠️ | RISKY | 可能遺失未支援的功能 |
| 修改複雜文件 | ❌ | UNSAFE | roundtrip 會損壞 VBA/ActiveX/SmartArt/Diagrams |
| Track Changes | ❌ | 不支援 | 用 OOXML 或 Word COM |
| TOC ���成 | ⚠️ | RISKY | 能插入 field code，但不能更新 |
| 交叉引用 | ❌ | 不支援 | 用 OOXML raw XML |
| 浮動圖片 | ❌ | 不支援 | 只有行內圖片 |

### ❌ CJK 字型只設一半
```python
# ❌ 錯——只設 Latin 字型，CJK 字會用預設字型
run.font.name = 'Arial'

# ✅ 對——同時設 Latin + CJK
run.font.name = 'Arial'
run._element.rPr.rFonts.set(qn('w:eastAsia'), '微軟正黑體')
```

### ❌ Style 直接 mutate
```python
# ❌ 錯——直接修改會汙染全域樣式
style.font.size = Pt(14)

# ✅ 對——copy 後新建
from copy import deepcopy
new_font = deepcopy(style.font)
```

### ❌ 橫向頁面不交換尺寸
```python
# ❌ 錯——只設方向，尺寸沒換，頁面會變形
section.orientation = WD_ORIENT.LANDSCAPE

# ✅ ��——方向 + 尺寸一起設
section.orientation = WD_ORIENT.LANDSCAPE
section.page_width = Cm(29.7)
section.page_height = Cm(21.0)
```

### ❌ 合併儲存格後不重設邊框
合併後只有左上角 cell 保留邊框設定，其餘被清空。必須對合併範圍的每個 cell 逐一設定邊框。

### ❌ 分節後頁首自動連結
新增分節後，頁首預設 `is_linked_to_previous = True`。如果不主動斷開，新分節會繼承前一節的頁首。必須在寫入內容**之前**設 `is_linked_to_previous = False`。

### ❌ python-docx roundtrip 損壞
開啟 → 修改 → 存檔的過程中，python-docx 會丟棄它不認識的 XML 元素：
- VBA macro
- ActiveX controls
- SmartArt / Diagrams
- Sparklines
- Digital signatures
- 自訂 XML parts

**對策**：碰複雜既有檔案 → 先跑 `assess_risk.py` → UNSAFE 強制切 Word COM。

## OOXML 地雷

### ❌ Track Changes 缺少必要屬性
```xml
<!-- ❌ 缺 w:id、w:author、w:date -->
<w:ins><w:r><w:t>文字</w:t></w:r></w:ins>

<!-- ✅ 完整 -->
<w:ins w:id="1" w:author="文本參謀長" w:date="2026-04-05T12:00:00Z">
  <w:r><w:rPr/><w:t xml:space="preserve">文字</w:t></w:r>
</w:ins>
```

### ❌ 刪除用 w:t 而非 w:delText
```xml
<!-- ❌ 錯——刪除的文字仍會顯示 -->
<w:del><w:r><w:t>被刪文字</w:t></w:r></w:del>

<!-- ✅ 對 -->
<w:del><w:r><w:delText>被刪��字</w:delText></w:r></w:del>
```

### ❌ w:id 不唯一
每個 `w:ins`、`w:del`、`w:comment`、`w:bookmark` 的 `w:id` 必須全文件唯一。重複 ID 會導致 Word 無法正確顯示修訂。

### ❌ 忘記 xml:space="preserve"
```xml
<!-- ❌ 前後空格會被吃掉 -->
<w:t> 有空格的文字 </w:t>

<!-- ✅ 保留空格 -->
<w:t xml:space="preserve"> 有空格的文字 </w:t>
```

### ❌ Smart Quotes 編碼
在 XML 中必須用 entity：
- `"` → `&#x201C;`（左雙引號）
- `"` → `&#x201D;`（右雙引號）
- `'` → `&#x2018;`（左單引號）
- `'` → `&#x2019;`（右單引號）

直接寫 Unicode 字元在某些解析器中會出問題。

### ❌ Namespace 遺漏
OOXML 有 30+ 個 namespace。編輯 XML fragment 時必須確保所有用到的 namespace 都有宣告。缺少 namespace 宣告 = Word 無法開啟文件。

## Word COM 地雷

### ❌ 忘記關閉 Word process
```python
# ��� 錯——Word process 殘留，佔記憶體，鎖住檔案
word = win32com.client.Dispatch('Word.Application')
doc = word.Documents.Open(path)
# ... 中途出錯，word.Quit() 沒被呼叫

# ✅ 對——永遠用 try/finally
word = win32com.client.Dispatch('Word.Application')
word.Visible = False
try:
    doc = word.Documents.Open(path)
    # ... 操作 ...
    doc.Close(SaveChanges=True)
finally:
    word.Quit()
```

### ❌ PDF 用 SaveAs 而非 ExportAsFixedFormat
```python
# ❌ 品質較差，某些格式遺失
doc.SaveAs(path, FileFormat=17)

# ✅ 最高品質 PDF
doc.ExportAsFixedFormat(
    OutputFileName=path,
    ExportFormat=17,
    OptimizeFor=0,
    CreateBookmarks=1
)
```

### ❌ COM 物件路徑不接受正斜線
```python
# ❌ Word COM 不認正斜���
doc = word.Documents.Open('C:/Users/USER/file.docx')

# ✅ 反斜線或 raw string
doc = word.Documents.Open(r'C:\path\to\file.docx')
```

### ❌ 操作中檔案被其他程式鎖住
Word COM 開啟檔案時會取得獨占鎖。如果檔案已在 Word 中開啟，`Documents.Open()` 會失敗。

**對策**：操作前檢查是否有 WINWORD.EXE process 佔用該檔案。

## 格式轉換地雷

### ❌ Pandoc Markdown → DOCX 不帶 reference-doc
```bash
# ❌ 輸出用 Pandoc 預設樣式，很醜
pandoc input.md -o output.docx

# ✅ 套用模板樣式
pandoc input.md -o output.docx --reference-doc=template.docx
```

### ❌ pdf2docx 對掃描型 PDF 無效
pdf2docx 只能處理文字型 PDF。掃描型（圖片）PDF 需要先 OCR：
```bash
# 先 OCR
tesseract input.pdf output pdf
# 再轉換
```

### ❌ Pandoc 路徑含中文
某些 Pandoc 版本在 Windows 上對含中文的路徑會出錯。**對策**：操作前將檔案複製到純英文路徑。
