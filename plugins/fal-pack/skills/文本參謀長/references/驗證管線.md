## 三層驗證架構

### Layer 1：結構驗證（scripts/verify_structure.py）

**自動化驗證項目**：

```python
def verify_structure(docx_path, design_system=None):
    """
    結構驗證：不開 Word，純 python-docx 分析。
    回傳 (pass: bool, issues: list[str])
    """
    doc = Document(docx_path)
    issues = []
    
    # 1. 標題層級連續性
    heading_levels = []
    for p in doc.paragraphs:
        if p.style.name.startswith('Heading'):
            level = int(p.style.name.split()[-1])
            heading_levels.append(level)
    
    for i in range(1, len(heading_levels)):
        if heading_levels[i] > heading_levels[i-1] + 1:
            issues.append(f"標題跳級: H{heading_levels[i-1]} → H{heading_levels[i]}")
    
    # 2. CJK 字型雙設定
    for p in doc.paragraphs:
        for run in p.runs:
            if has_cjk(run.text):
                rFonts = run._element.rPr.rFonts if run._element.rPr else None
                if not rFonts or not rFonts.get(qn('w:eastAsia')):
                    issues.append(f"CJK 字型未設定: '{run.text[:20]}...'")
    
    # 3. 樣式一致性（如果有 Design System）
    if design_system:
        # 抽查字型、字級、顏色是否符合 Design System
        pass
    
    # 4. Metadata 完整性
    props = doc.core_properties
    if not props.author:
        issues.append("Metadata 缺少 author")
    if not props.title:
        issues.append("Metadata 缺少 title")
    if not props.language:
        issues.append("Metadata 缺少 language")
    
    # 5. 無障礙檢查
    # 圖片 alt text
    for rel in doc.part.rels.values():
        if "image" in rel.reltype:
            # 檢查對應的 docPr 是否有 descr
            pass
    
    # 表格標頭
    for table in doc.tables:
        first_row = table.rows[0]
        # 檢查 tblHeader 屬性
        tr = first_row._tr
        trPr = tr.find(qn('w:trPr'))
        if trPr is None or trPr.find(qn('w:tblHeader')) is None:
            issues.append("表格缺少 header row 標記")
    
    # 6. 交叉引用有效性
    # 掃描 field code，確認引用的 bookmark 存在
    
    return (len(issues) == 0, issues)
```

### Layer 2：視覺品檢（scripts/render_preview.py）

**Word COM → PDF → PNG**，表弟自己讀 PNG 做視覺檢查。

```python
def render_preview(docx_path, output_dir):
    """
    產出視覺預覽圖。
    1. Word COM 開啟 → 匯出 PDF
    2. PyMuPDF 將 PDF 轉為 PNG（每頁一張）
    3. 回傳 PNG 路徑列表
    """
    import win32com.client
    import fitz  # PyMuPDF
    
    word = win32com.client.Dispatch('Word.Application')
    word.Visible = False
    
    try:
        doc = word.Documents.Open(os.path.abspath(docx_path))
        pdf_path = os.path.join(output_dir, 'preview.pdf')
        doc.ExportAsFixedFormat(
            OutputFileName=pdf_path,
            ExportFormat=17,
            OptimizeFor=0,
            CreateBookmarks=1
        )
        doc.Close(SaveChanges=False)
    finally:
        word.Quit()
    
    # PDF → PNG
    pdf = fitz.open(pdf_path)
    png_paths = []
    for i, page in enumerate(pdf):
        pix = page.get_pixmap(dpi=150)
        png_path = os.path.join(output_dir, f'page_{i+1:03d}.png')
        pix.save(png_path)
        png_paths.append(png_path)
    pdf.close()
    
    return png_paths
```

**表弟檢查清單**：
- [ ] 文字有沒有被截斷/遮蔽
- [ ] 表格有沒有溢出頁面
- [ ] 圖片位置正確嗎
- [ ] 頁首頁尾顯示正確嗎
- [ ] 頁碼連續嗎
- [ ] 斷行合理嗎（特別注意 CJK 避頭尾）
- [ ] 整體視覺平衡——沒有「頭重腳輕」

```python
def check_compatibility(docx_path):
    """
    檢查 Google Docs 相容性和跨平台問題。
    """
    issues = []
    
    # 1. 字型相容性
    safe_fonts = {
        'latin': ['Arial', 'Times New Roman', 'Calibri', 'Georgia'],
        'cjk': ['Noto Sans TC', 'Noto Sans SC', 'Noto Sans JP',
                '微軟正黑體', 'Microsoft JhengHei'],
    }
    # 掃描所有 run 的字型，標記不在安全清單的
    
    # 2. 功能相容性
    # ZIP 層級掃描不相容的功能
    with zipfile.ZipFile(docx_path, 'r') as z:
        names = z.namelist()
        if 'word/vbaProject.bin' in names:
            issues.append("含 VBA macro — Google Docs 不支援")
        if any('activeX' in n for n in names):
            issues.append("含 ActiveX — Google Docs 不支援")
        if any('charts' in n for n in names):
            issues.append("含內嵌圖表 — Google Docs 可能顯示異常")
    
    # 3. PDF 輸出測試
    # 用 Word COM 匯出 PDF，確認無錯誤
    
    return (len(issues) == 0, issues)
```

## 驗證流程

```
產出 docx
  │
  ▼
Layer 1: 結構驗證
  ├─ PASS → 繼續
  └─ FAIL → 自動修復可修的 → 重驗 → 仍 FAIL → 停下報告
  │
  ▼
Layer 2: 視覺品檢
  ├─ PASS → 繼續
  └─ FAIL → 定位問題 → 修復 → 重新渲染 → 仍 FAIL → 停下報告
  │
  ▼
Layer 3: 相容性閘門
  ├─ PASS → 全過，交付
  └─ FAIL → 標記不相容項目
       ├─ 可修復（字型替換等）→ 修 → 重驗
       └─ 不可修復（功能限制）→ 告知使用者取捨
```

---

## 「驗收」vs「體檢」

| | 驗收 | 體檢 |
|--|------|------|
| **範圍** | 三層全跑 | 只跑 Layer 1 |
| **視覺品檢** | 有 | 跳過 |
| **相容性** | 有 | 跳過 |
| **產出** | 通過/不通過 + 完整報告 | 健康度報告 |
| **適用** | 交付前最終檢查 | 快速了解文件狀態 |

---

## 自動修復能力

| 問題 | 自動修復 |
|------|---------|
| 標題跳級 | 降級到正確層級 |
| CJK 字型未設定 | 用 Design System 預設補上 |
| Metadata 缺失 | 用預設值填入（標記為需確認） |
| 缺 alt text | 標記（需人類補充，不自動生成） |
| 表格缺 header 標記 | 自動標記第一行 |
| 字型不相容 | 替換為安全清單中最接近的 |
