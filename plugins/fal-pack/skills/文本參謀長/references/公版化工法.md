# 公版化工法

> 讀取時機：使用者選擇「公版化」指令時載入。

---

## 什麼是公版化

把一份完成的文件**結晶為可複用的模板**——任何人拿到這個模板，換上自己的內容，就能產出同等品質的文件。

公版化 ≠ 存檔。公版化是把「設計決策」和「內容」分離，讓設計決策可以被複用。

---

## 公版化流程

### Step 1：辨識靜態區與動態區

- **靜態區**：不隨內容改變的部分——頁面設定、樣式定義、頁首頁尾、封面版型、品牌元素
- **動態區**：每次都會換的部分——標題、作者、日期、正文內容、數據、圖片
- **半靜態區**：結構固定但內容變——章節標題（可能微調）、表格框架（資料換但格式不換）

### Step 2：佔位符標記

動態區用 Jinja2 佔位符替換：

```
{{ title }}              → 文件標題
{{ author }}             → 作者
{{ date }}               → 日期
{{ executive_summary }}  → 執行摘要

{% for section in sections %}
    {{ section.heading }}
    {{ section.content }}
{% endfor %}

{% if has_appendix %}
    {{ appendix_content }}
{% endif %}
```

### Step 3：Design System 抽取

從完成的文件中提取所有格式定義，結晶為 `design_system.py`：

```python
DESIGN_SYSTEM = {
    'font_latin': 'Times New Roman',
    'font_cjk': '微軟正黑體',
    'size_h1': 18,
    'size_body': 11,
    # ... 完整定義
}
```

### Step 4：模板規格文件

產出 `template_spec.yaml`：

```yaml
name: "商業企劃書"
version: "1.0"
created: "2026-04-05"
description: "標準商業企劃書模板，含執行摘要/問題/方案/財務"

placeholders:
  - name: title
    type: string
    required: true
    description: "企劃書標題"
  - name: sections
    type: list
    required: true
    description: "章節列表，每項含 heading + content"
    schema:
      heading: string
      content: rich_text

design_system: "design_system.py"
template_file: "template.docx"
```

### Step 5：驗證

用測試資料填充模板，跑驗證管線確認：
- 佔位符全部被替換（無殘留 `{{`）
- 格式正確
- 動態內容長度變化時排版不壞

### Step 6：歸檔

```
templates/
├── [模板名稱]/
│   ├── template.docx        # Word 模板
│   ├── design_system.py     # 格式定義
│   ├── template_spec.yaml   # 規格文件
│   └── sample_output.pdf    # 範例產出（用測試資料）
```

---

## 公版化的品質標準

- [ ] 所有動態區都有佔位符
- [ ] 所有靜態區都被鎖定（不會被 docxtpl 誤觸）
- [ ] Design System 完整——從模板生成的文件不需要額外的格式設定
- [ ] 規格文件清楚——全新的 fal 只讀規格就能正確使用模板
- [ ] 測試資料生成的樣本通過驗證管線

---

## 變動地圖

每個模板必須有變動地圖：

| 區域 | 變動頻率 | 變動觸發 | 處理方式 |
|------|---------|---------|---------|
| 靜態區 | 不變 | — | 鎖死 |
| 動態區 | 每次 | 新內容 | docxtpl 填充 |
| Design System | 低頻 | 品牌更新/優化 | 更新 design_system.py |
| 骨架結構 | 低頻 | 內容方更新契約 | 重建模板 |
