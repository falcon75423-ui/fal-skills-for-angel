## 核心原則（靜態區）

- **同一個問題至少兩個不同武器搜**——它們的來源集合幾乎不重疊
- **語意層 + 廣度層為雙引擎核心**——目前是 Exa + WebSearch，未來工具換了原則不變
- **武器不到位就別出發**——測繪時做武器就緒檢查，取不到的回報問題，不跳過
- **武器×心法雙綁**——不堆清單，看維度。每個維度知道用什麼武器組合 + 怎麼用。維度視角見下方「武器×心法雙綁」段

## 武器×心法雙綁（4 個維度）

> 維度視角看武器：什麼情境用什麼維度的武器組，配合什麼心法。對應到品質下限的關係見每段末。

### 維度 1：精準度（語意精準 vs 廣度搜尋）

**武器組**：
- 語意精準：**Exa**（語意搜尋）+ **NotebookLM**（學術精準）
- 廣度搜尋：**WebSearch**（即時廣度）+ **Tavily**（特定網站爬蟲）

**心法**：
- 主題明確、要找具體論點 → 語意精準（Exa 為主）
- 主題探索、要建立地景 → 廣度搜尋（WebSearch 為主）
- 兩者必須交叉——同一個問題用語意 + 廣度雙軸搜
- Exa 進階參數：`category` 限定類型、`livecrawl: "preferred"` 強制最新

**對應品質下限**：**反面 / 弦外之音被跳過**——廣度太窄會錯過反面 + 弦外之音。每個主題至少一輪廣度搜尋打底。

---

### 維度 2：時效（即時 / 歷史 / 學術延遲）

**武器組**：
- 即時：**WebSearch**（內建零延遲） + **Exa** `livecrawl: "preferred"`
- 歷史：**Tavily** `start_date` / `end_date`（YYYY-MM-DD，唯一精確日期搜尋）
- 學術延遲：**NotebookLM**（論文索引可能延遲一兩週）

**心法**：
- 「上週發生了什麼」→ 即時武器
- 「2025-Q3 的事件演化」→ 歷史武器（Tavily date filter）
- 「最新研究結果」→ 學術武器（NotebookLM 為主，但要知道索引延遲）
- 時效標注：搜出來的資料必須帶日期，繳令時呈現「資料時間範圍」

**對應品質下限**：**失敗靜默繞過**——某時段拿不到資料時，必須記錄「該時段失敗」，不能用其他時段的資料替代。

---

### 維度 3：來源型（學術 / 一般 / 社群）

**武器組**：
- 學術：**NotebookLM**（deep mode 40 源） + **Exa** `category: "research paper"`
- 一般：**Exa** + **WebSearch** 雙引擎
- 社群：**WebSearch**（Reddit 主力）+ **Bash curl**（Reddit `.json` 留言）+ **Exa**（HN `site:news.ycombinator.com`）

**心法**：
- 學術：精度高但延遲，引用嚴謹的場景必走（決策有學術依據需求）
- 一般：日常主力，雙引擎覆蓋面廣
- 社群：聽現場聲音、找一手經驗、追熱點討論——但要警惕情緒、雜訊、迷因
- 三類來源不可互相取代——學術不能用社群替代、社群不能用學術替代
- 跨平台搜尋見 `~/.claude/references/platform-search-playbook.md`

**對應品質下限**：**維度被跳過**——三類來源容易漏社群（最容易被當「雜訊」跳過），但社群常有未報導的一手經驗。

---

### 維度 4：操作型（搜尋 / 精讀 / 爬蟲）

**武器組**：
- 搜尋：**Exa** / **WebSearch** / **NotebookLM**
- 精讀：**WebFetch** / **NotebookLM** `notebook_query`（單源做深度問答）
- 爬蟲：**Tavily** `tavily_map`（看網站結構） → `tavily_crawl`（精準爬取）

**心法**：
- 搜尋找線索 → 精讀深入 → 爬蟲系統性掃網站
- 同一個來源不必三層都做，按需求選層級
- WebFetch 打不開 → Tavily `extract` `advanced` 模式（受保護網站）
- 爬蟲耗用量大，限月度配額場景

**對應品質下限**：**單一來源被當高信心**——只用搜尋層拿到一個來源就停 = 沒做精讀層驗證 = 容易誤判。重要結論至少跑搜尋+精讀兩層。

## 武器清單（動態區）

### Exa — 🔴 必經武器（語意層）

語意搜尋品質顯著優於關鍵字搜尋，跳過它等於放棄最好的武器。取不到 → 回報問題，不跳過。

| 參數 | 用法 | 效果 |
|------|------|------|
| `category` | `"research paper"` / `"company"` / `"people"` | 精準限定搜尋類型，大幅提升訊噪比 |
| `livecrawl` | `"preferred"` | 強制抓取最新內容，繞過陳舊快取 |
| `numResults` | 預設 8，可調高 | 需要更多來源時調整 |
| `contextMaxCharacters` | 預設 10000 | 控制每筆結果的內容長度 |

**精搜場景**：
- 搜學術論文 → `category: "research paper"`
- 搜公司資訊 → `category: "company"`
- 搜人物背景 → `category: "people"`（LinkedIn 等）
- 需要最新資料 → `livecrawl: "preferred"`

### WebSearch — 廣度層

即時新聞、最新動態、廣泛覆蓋。零延遲（內建於 API 呼叫）。

支援 `allowed_domains` / `blocked_domains` 做域名篩選。

### NotebookLM — 學術層 + 持久情報庫

學術論文、arXiv、深度綜述、交叉引用。

**進階能力**：
- `research_start` 的 `deep` 模式：5 分鐘內自動找到 ~40 個來源並匯入持久性筆記本
- `cross_notebook_query`：跨筆記本查詢——對同一主題做過多次研究時，可以交叉查詢歷史情報
- 持久性：研究結果存在 NotebookLM 筆記本裡，下次相關主題不用從零開始

### WebFetch — 精讀層

特定 URL 深讀 + AI 摘要。已鎖定的重要來源深入閱讀用。不能搜尋，只能讀。

### Tavily — 備用武器（有月度用量限制）

有三個獨特能力是其他武器做不到的，但日常偵察中主力武器已足夠覆蓋。用量有限時優先留給這些場景：

| 獨特能力 | 說明 | 用於 |
|---------|------|------|
| **日期篩選** | `start_date` / `end_date`（YYYY-MM-DD），唯一支援精確日期範圍的搜尋 | 「某段時間內發生了什麼」的時序研究 |
| **偵察式爬蟲** | `tavily_map`（先看網站結構）→ `tavily_crawl`（再精準爬取） | 系統性分析特定網站 |
| **保護性網站擷取** | `tavily_extract` 的 `advanced` 模式可處理 LinkedIn 等受保護網站 | WebFetch 打不開的來源 |

## 武器搭配建議

| 研究類型 | 主武器 | 副武器 | 備註 |
|---------|--------|--------|------|
| 技術選型 | Exa + WebSearch | NotebookLM（方法論） | Exa 用 `category: "research paper"` 搜技術論文 |
| 市場研究 | Exa + WebSearch | NotebookLM（產業報告） | Exa 用 `category: "company"` 搜公司資訊 |
| 學術研究 | NotebookLM（deep）+ Exa | WebFetch（特定論文精讀） | NotebookLM deep 模式一次拿 40 源 |
| 時事追蹤 | WebSearch | Exa（`livecrawl: "preferred"`） | WebSearch 為主，即時性優先 |
| 人物調研 | Exa（`category: "people"`）+ WebSearch | Tavily extract（LinkedIn） | |
| 時序研究 | Tavily（日期篩選）+ WebSearch | Exa（背景脈絡） | Tavily 的獨特場景 |
| 特定網站深挖 | Tavily（map → crawl） | WebFetch（重點頁面精讀） | Tavily 的獨特場景 |
