# 已知地雷

> 🔴 每次偵察啟動時強制載入。偵察兵的免疫系統。
>
> 跟 `_index.md` 「品質校準」段對照看：地雷是**動作層**（具體操作怎麼避免踩雷），品質下限 / 警戒是**驗收層**（任務後檢核哪條被觸碰）。同一個風險在兩個檔案以兩種視角寫，互補不重複。

---

## 搜尋工具地雷

❌ **Exa 不可用時靜默跳過，只用 WebSearch 完成任務**
✅ Exa 是必經武器。取不到 → 停下來回報問題，不自行降級。Exa 和 WebSearch 的來源集合幾乎不重疊，少一個引擎 = 系統性盲區

❌ **WebFetch 打不開的頁面，就放棄那個來源**
✅ 先試 Tavily `extract`（`advanced` 模式可處理受保護網站）。真的打不開 → 標記為「🔴 無法存取」，不靜默丟棄

❌ **NotebookLM `research_start` 回傳後直接用，不檢查來源品質**
✅ `research_start` 自動匯入的 ~40 個來源品質參差不齊。必須過濾後才能引用——NotebookLM 是來源收集器，不是品質保證

❌ **搜尋結果的 URL 不驗證就引用**
✅ 搜尋引擎（包括 Exa）有時回傳過期、重定向、或已刪除的 URL。關鍵引用必須用 WebFetch 驗證頁面實際存在

## 流程地雷

❌ **搜尋輪次之間不壓縮，把所有原始結果帶到下一輪**
✅ 每輪結束時壓縮為精華摘要。下一輪只帶：原始問題 + 累積精華 + 上一輪結果。塞太多反而變笨（認知窒息）

❌ **覆蓋率不足時，自動放行並交付「部分完成」的報告**
✅ 覆蓋率不夠就繼續搜。停的判斷是「情報夠做決策了」而非「時間到了」。真的搜不到 → 寫進「我不知道」清單，不假裝覆蓋

❌ **子 Agent 之間共享 context 或互相看結果**
✅ 研究者隔離是硬規則。獨立調查 → 監督者彙總。共享 context 導致群體思維

❌ **模型訓練知識靜默替代搜尋結果**
✅ 如果用了模型知識，必須標注「⚠️ 此段基於模型知識，非即時搜尋」。否則下游無法判斷時效性

## 品質地雷

❌ **單一來源就給 🟢 高信心標注**
✅ 單一來源永遠最高 🟡。🟢 需要至少兩個獨立來源交叉驗證

❌ **引用來源「充數」——塞不相干的來源撐數量**
✅ 每個引用必須真正支持它所附著的主張。來源的價值在必要性，不在數量

❌ **偵察計畫訂了就照走，不管搜到的東西暗示什麼**
✅ 偵察計畫是活的。每輪結束時重規劃：有新線索值得追嗎？有維度已飽和可以關閉嗎？
