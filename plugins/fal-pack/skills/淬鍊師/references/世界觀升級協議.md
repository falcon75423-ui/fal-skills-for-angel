# 世界觀升級協議

> 讀取時機：教訓從 Macro 升級到現場主人 SKILL.md 世界觀區塊前必經。
>
> 核心：**沒有協議，世界觀升級只是規則層補丁的 meta 版**——下次同類錯誤會繞過去。

---

## 為什麼要這個協議

沒有協議時，世界觀升級 = 在 SKILL.md 加一句話。

🔴 **這就是規則層錯層陷阱的 meta 版**（見退役 M02 層級錯配必失效）。下次同類錯誤發生時，那句話擋不住，因為它只是靠執行者記得看。

協議強迫每次升級都配備**上下游結構**，讓世界觀從「告誡」變成「活的防護」。

---

## 觸發條件

教訓累積到任一信號出現時，進入本協議：

| 信號 | 意義 |
|------|------|
| 錯誤類型累計 ≥ 3 次（Three Sightings） | 教訓該升級為原則或世界觀 |
| 現場主人 SKILL.md 閘門復發 ≥ 2 次 | 閘門失效，禁用「擴條件」，走三選一（退役/降級/重問 WHO）|
| 既有世界觀條目**顯性觸發**本次失效 | 世界觀需補結構配套（上游文字有、結構沒跟上）|
| 新的跨 Skill 通用模式浮現 | 考慮鍛造新的通用機制 |

任一觸發 → 進入五題 checklist。

---

## 五題 checklist

教訓升級為 SKILL.md 世界觀區塊前，必答五題。**任一題答不出來 = 升級不成立，補完再升**。

### 第 1 題：具象失效場景

> 過去哪次真實事件最能體現這條世界觀的缺失？舉**至少一個具體案例**。

**為什麼**：世界觀必須有真實失效背書。沒有具象失效 = 概念太抽象 = 還沒成熟到可以升級。

**不通過的信號**：
- 「常常會發生」「我覺得有時候」（抽象敘述）
- 舉不出具體案件編號 / 日期 / 對話片段
- 例子是「未來可能會」而不是「過去確實發生」

**補強方向**：
- 翻 lessons / 鍛造記錄 / 工作簿，找真實案件
- 舉不出來 → 該世界觀還不成熟，先回去當教訓累積

### 第 2 題：上游位置

> 世界觀文字寫在 SKILL.md 哪個區塊？（世界觀區 / 人格 / 原則 / 盲點 / 身份區塊）

**為什麼**：放錯區塊 = 規則層失效。「世界觀」放在「原則」會被當成可協商的，「人格」放在「盲點」會被當成需要修的——位置定義承諾強度。

**不通過的信號**：
- 「先放某處再說」「等 LLM 自己判斷」
- 沒考慮過放哪
- 想放好幾個地方都覺得對 = 沒想清楚它跟現有系統的關係

**補強方向**：
- 對照 SKILL.md 既有結構，找最自然的位置
- 兩個位置都對 → 拆成兩條條目，分別放對應位置

### 第 3 題：中游結構

> 哪個可執行的結構體現這條世界觀？（檔案 / 骨架 / 對照表 / 閘門 / 硬規則 / 字典 / 追蹤表）

**為什麼**：只有文字沒結構 = 規則層病。下次靠記憶力仍會失效——這是 M02 層級錯配的本質。

**不通過的信號**：
- 「LLM 看到自然會做」「應該會記得」
- 結構是「在 SKILL.md 加一段提醒」（提醒 ≠ 結構）
- 結構沒形式觸發點（必須靠主觀判斷啟動）

**補強方向**：
- 把「告誡」轉為「形式自動觸發」（如字面字串檢測 / checklist 條目 / 表格欄位）
- 找姊妹結構參考（如拍板訊息骨架的「字面 Q1-QN 觸發」設計）

### 第 4 題：下游偵測

> 什麼機制偵測這條世界觀被違反？（踩雷錄專欄 / 活躍症狀追蹤 / 閘門失效追蹤 / Three Sightings 計數表 / 硬規則計數器）

**為什麼**：違反了不會響鈴 = 無反饋 = 飛輪斷鏈。世界觀失效時必須能被偵測，不然下次同類錯誤會悄悄發生而沒人知道。

**不通過的信號**：
- 「LLM 自己會發現」「下次就改進」
- 沒有專欄 / 表格 / 計數機制
- 偵測機制依賴主觀判斷（「覺得不對」）

**補強方向**：
- 在踩雷錄加一欄
- 在閘門失效追蹤表加一行
- 設計「字面 grep 可掃」的偵測（最強）

### 第 5 題：回測驗證

> 用第 1 題的具象失效場景回測，第 2+3+4 的結構能擋住嗎？

**為什麼**：結構擋不住失效 = 沒對準 = 白做。回測是「裝完防護後拿原始失效再來一次」的閉環驗證。

**不通過的信號**：
- 「應該擋得住」「理論上可以」
- 回測時要靠主觀判斷才能擋住
- 回測時發現結構繞得過去

**補強方向**：
- 回去補強結構（重做第 3 題）
- 回去補強偵測（重做第 4 題）
- 兩個都補了還擋不住 → 該世界觀的根因可能不對，回 lessons 再淬鍊

---

## 五題 checklist 全通過後

✅ 通過 = 升級成立 → 進入後續行動：

1. **寫進對應位置**：把世界觀條目寫進 SKILL.md 第 2 題指定的區塊
2. **建立中游結構**：實際建立第 3 題的結構（不是「規劃要建」，是真的建）
3. **接通下游偵測**：實際在偵測機制加一行/一欄
4. **補做三情境衝撞**：五題第 5 題僅單情境，跟 `references/三情境衝撞工法.md` 的衝撞配套使用
5. **退役舊條目**：原 lessons 條目標退役（在 ARCHIVE.md），frontmatter 加 `retired_at` + `retired_reason: 升級為 SKILL.md 世界觀`

---

## 升級失敗的處理

某題答不出來時的處理：

| 失敗題 | 處理 |
|--------|------|
| 第 1 題（具象失效）答不出來 | **不升級**——回去當 lessons 教訓累積，等更多案件樣本 |
| 第 2 題（上游位置）答不出來 | **暫停**——召喚淬煉團「結構審視者」討論該放哪 |
| 第 3 題（中游結構）答不出來 | **暫停**——召喚淬煉團「邊界挑戰者 + 簡化批判者」設計結構 |
| 第 4 題（下游偵測）答不出來 | **暫停**——召喚淬煉團「策略偵察者」設計偵測機制 |
| 第 5 題（回測）失敗 | **回去重做 3+4**——結構或偵測沒對準失效模式 |

🔴 **絕對紀律**：失敗的題沒補完不得跳過入庫。失敗 = 升級不成立 = 該世界觀不能進 SKILL.md。

## 落地實例：F-001 白話閘門退役為四件套

時間：2026-04-24
案件：白話閘門連續復發 2 次 → 觸發閘門失效追蹤的第 6 條硬規則 → 進入世界觀升級協議

| # | 答案 |
|---|------|
| **1 具象失效** | 2026-04-23 N-027 + 2026-04-24 N-028 白話閘門連續復發（技術名詞 fallback / grep / glob / schema / docstring 脫口而出，使用者看不懂無法拍板）|
| **2 上游位置** | SKILL.md 身份區塊「講話的預設姿態」 |
| **3 中游結構** | 拍板訊息骨架（形式強制，Q1-QN / A/B/C 字面觸發）+ `references/使用者語彙.md`（對照表）|
| **4 下游偵測** | 踩雷錄「閘門失效追蹤」專欄 + 教訓索引運作規則第 6 條（閘門復發硬規則）|
| **5 回測** | ✅ 若有這套結構在 N-027 之前就位，該次訊息必走拍板骨架 → 技術名詞必查字典 → 字典有翻譯必翻譯 → 失效被擋 |

**這是第一個通過五題 checklist 的世界觀升級實例**。未來每一次升級都應產出類似的五題答案，存到對應的 Bridge Document 裡。

## 跟三情境衝撞的關係

世界觀升級協議五題 = **三情境衝撞驗證在「上游世界觀」層的具體版**：
- 五題的前 4 題提供結構配套（具象失效 / 上游位置 / 中游結構 / 下游偵測）
- 五題的第 5 題「回測驗證」= 三情境衝撞的單情境簡化版

**完整流程**：

```
觸發信號（Three Sightings / 閘門復發 / 顯性觸發 / 跨 Skill 通用模式）
  ↓
進入五題 checklist
  ↓
五題全通過 → 補做三情境衝撞（邊界 + 衝突 + 反例）
  ↓
三情境全通過 → 升級成立 → 寫進 SKILL.md
```

🔴 **不能只跑五題不跑三情境**——五題第 5 題只是單情境回測，未涵蓋邊界 / 衝突 / 反例三類張力。

> **三情境衝撞詳細工法** → `references/三情境衝撞工法.md`

## 工法的工法（meta 層）

世界觀升級協議本身——

- **R1 自審**：5 題的順序對嗎？（具象→位置→結構→偵測→回測 = 由具體到抽象的升級鏈）
- **R2 自審**：失敗處理規則明確嗎？（每題失敗對應不同處理路徑，已寫表）
- **R3 自審**：協議會不會被當作「升級的形式手續」而不是真的把關？（風險存在，靠淬鍊師「不允許跳過」紀律）
- **R4 自審**：誰會用這個協議？（淬鍊師判斷觸發 → 主動進入協議 → 自己答 5 題或召喚淬煉團補強）

**結論**：本協議 R1-R4 自審通過，可投入使用。本協議本身已有 F-001 一個落地實例驗證——通過實戰回測。
