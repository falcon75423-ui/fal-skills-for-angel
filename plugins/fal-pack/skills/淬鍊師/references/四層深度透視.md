# 四層深度透視

> 讀取時機：路徑 B（淬煉一個 md 檔）必讀。其他路徑遇到「需要切多層分析」時也讀。
> SKILL.md 「工法路徑」段會指引到這裡。

四層深度透視是淬鍊師對既有智慧資本做手術時用的解剖刀——**從骨架到靈魂，由淺入深**。

---

## 為什麼是 4 層

不是 3 層（不夠細）、不是 5 層（會增加摩擦但不增加洞察）、不是 6 層（六色帽的無限論辯陷阱）。

4 層的選定來自實戰：**結構 / 規則 / 策略 / 動機** 四個失效類別在 F-001 ~ F-004 的鍛造案件中反覆出現，**任一層的問題其他層都無法替代回答**——所以是 4 個獨立的觀察維度。

| # | 層 | 物理位置 | 一句話判別 |
|---|----|---------|-----------|
| R1 | **結構層** | 骨架 | 拓撲對嗎？因果鏈閉環嗎？熵值會增加還是減少？ |
| R2 | **規則層** | 血肉 | 邊緣 case 處理了嗎？什麼情境會炸？ |
| R3 | **策略層** | 行為 | 真的解決問題還是在藏延遲？代價在哪？ |
| R4 | **動機層** | 靈魂 | 使用者真的需要這個嗎？做完還像個人會用的嗎？ |

🔴 **順序原則**：先淺後深。骨架歪了不必修血肉、血肉錯了不必問動機。**任一層發現問題就停下來修，不必跑完 4 層**。

---

## R1 結構層

### 底層邏輯
拓撲學 + 因果鏈 + 熵值

### 判別問題
1. **責任鏈閉環嗎？** 從輸入到輸出的責任鏈是否完整？有沒有「中途斷掉沒人接」的洞？
2. **拓撲對嗎？** 元素之間的依賴關係（誰需要誰）是否正確？有沒有循環依賴？
3. **熵值方向？** 這個結構會讓系統變更整齊還是更混亂？
4. **層級正確嗎？** 應該在 SKILL.md 的東西在 references 裡？應該在通用層的東西塞進產品層？

### 工法步驟
1. 畫出元素清單（用條列）
2. 畫出元素之間的依賴關係（誰指向誰）
3. 找閉環斷點（有沒有元素「沒人接」或「無限循環」）
4. 找層級錯配（穩定度不同的東西混住）

### md 精煉應用

對 md 檔做 R1 透視：
- **段落順序**：開頭應該先講什麼？目前順序是不是「需要先讀後段才懂前段」？
- **資訊分層**：SKILL.md 必讀的、references 詳述的、附錄考古的——分層對嗎？
- **指針有效性**：md 裡的 `→ references/X.md` 指針指到的檔案存在嗎？
- **重複內容**：同一個事實是不是寫在 A 和 B 兩處（E10 雙份文件反例）？

### 失效訊號
- 讀者要往回翻才懂某個術語的定義
- 同一個概念在兩個地方有不同說法
- 「為什麼」和「怎麼做」混在同一段
- 出現「按照之前的格式」這種隱式依賴

---

## R2 規則層

### 底層邏輯
墨菲定律執行者

### 判別問題
1. **最痛苦的路徑長怎樣？** 「正常情況」之外的邊緣會怎樣？
2. **使用者中途中斷會怎樣？** 中途關瀏覽器、按 Ctrl+C、斷網——系統穩嗎？
3. **重複觸發呢？** 同一個動作做兩次會累積副作用嗎？（idempotent 嗎）
4. **空值/極值呢？** null、空字串、超大值、負數、零——處理了嗎？

### 工法步驟
1. 列出快樂路徑
2. 對每一步問「這一步失敗會怎樣」
3. 找警示力不足的條款（如「應該」「通常」「大部分時候」這些字眼 = 沒明確定義邊界）
4. 找「未實作」的描述（描述式紀律 = 看到就忘）

### md 精煉應用

對 md 檔做 R2 透視：
- **「應該」「通常」掃描**：grep `應該|通常|大部分|一般來說` → 每一條都問「邊緣 case 是什麼」
- **隱性閘門**：「下次 X 自動 Y」「會自動觸發」這類字樣 = 警報（見 #039 教訓「規格依賴隱性鉤子必然失敗」）
- **缺失的 fallback**：每個「呼叫 X」都問「X 失敗時怎麼辦」
- **未明示的 unit**：「等一下」「短時間內」「比較久」——數字呢？

### 失效訊號
- 描述用「應該會」「大部分時候」這種模糊詞
- 動詞描述的步驟沒有對應 checklist 強制（描述式 ≠ 閘門式）
- 缺失 happy path 之外的所有狀態
- 「先 X 再 Y」沒寫「如果 X 失敗呢」

## R3 策略層

### 底層邏輯
賽局理論 + 資訊不對稱 + 過度工程偵測

### 判別問題
1. **代價呢？** 這個解法的隱藏成本在哪？延遲被藏起來了嗎？
2. **症狀還是病因？** 修的是表面現象還是根本原因？
3. **過度工程嗎？** YAGNI 違反——多做了下次根本不會用的東西？
4. **重工嗎？** 跟既有機制重複了嗎？（一進一出原則）

### 工法步驟
1. 列出本次新增的所有規則 / 結構 / 機制
2. 對每一條問「沒有它會死嗎」
3. 對每一條問「現有生態系裡有沒有已經能做這件事的」
4. 對每一條問「這是修補丁還是治本」

### md 精煉應用

對 md 檔做 R3 透視：
- **重複規則合併**：同一原則在 3 個地方被講 3 次 → 該層失效（見飛輪防錯第六條）
- **未升級的補丁**：補丁層的條款應該升級到結構層或閘門層的，被留在補丁層
- **過度規則化**：每個規則都加一條 enforcement → 讀者放棄閱讀（規則堆積 = 反淬鍊）
- **沒接線的描述**：寫了「我們會做 X」但實作端沒有任何閘門 / 腳本 / 檢查在做這件事（M08 反例）

### 失效訊號
- 條款 N+1、N+2、N+3 都在說同一件事（該升級到上一層）
- 「為了下次可能用得到」（YAGNI 違反）
- 描述跟實作脫鉤（S-INFO-07 自動化虛寫）
- 修補丁解決症狀，沒回去問為什麼會出現症狀

## R4 動機層

### 底層邏輯
HCI + 心理學 + 人類學

### 判別問題
1. **使用者真的需要這個嗎？** 還是只是設計者自嗨的技術堆疊？
2. **使用者帶著什麼煩惱來？** 我們解的是哪個煩惱？
3. **靈魂在哪？** 寫得精密但讀起來冷冰冰，這個東西做完還像個人會用的嗎？
4. **誰會在什麼時機打開這個檔案？** 沒人會打開的檔案 = 沒存在意義

### 工法步驟
1. 想像具體的使用者（不是抽象的「user」）—— 使用者/ 工頭 / 客戶 Skill / 下一個 fal
2. 對每一段問「誰會在什麼情境讀這段」
3. 對每一段問「讀完會做出什麼行動」
4. 對每一段問「換一個人來讀還成立嗎」

### md 精煉應用

對 md 檔做 R4 透視：
- **受眾對齊**：這檔是給模型讀的、給人讀的、還是給未來的自己讀的？三者寫法不同
- **打開時機**：使用者什麼時候會主動翻這檔？沒答案 = 該檔位置錯了
- **生命力校準**：讀起來有溫度嗎？還是像 ISO 標準書？知識載體不該失去呼吸
- **減法極限**：刪掉這段功能會壞嗎？答「不會」就刪

### 失效訊號
- 寫了一堆但沒人會去讀
- 受眾從沒被想清楚（「給所有人」= 給沒人）
- 完美的規格，零實際價值
- 「為了完整性」（Completeness 是 R3 過度工程的另一種包裝）

## 4 層 vs 淬煉團 5 角色

四層深度透視 = **單人淬鍊師的解剖工法**（成本低、適中游淬鍊）。
淬煉團 5 角色 = **多人多視角碰撞**（成本高、適上游 / 高張力）。

對應關係：

| 層 | 對應角色 | 何時用 4 層 vs 何時召喚 5 角色 |
|----|---------|----------------------------|
| R1 結構 | 結構審視者 | 簡單議題 → 4 層自跑；高張力 → 召喚 5 角色 |
| R2 規則 | 邊界挑戰者 | 同上 |
| R3 策略 | 策略偵察者 | 同上 |
| R4 動機 | 動機追問者 + 簡化批判者 | 5 角色版多了「簡化批判者」獨立刃，4 層版的 R4 兼任 |

**判別**：上游淬鍊（世界觀 / 人格 / 智慧層 / 高張力議題）→ 5 角色全進場。中游淬鍊（閘門 / 條目 / 流程 / md 檔）→ 4 層自跑通常夠用。

> **5 角色召喚規則** → `references/淬煉團召喚指南.md`

## 順序原則：先結構後動機

🔴 **R1 → R2 → R3 → R4 是預設順序**——先看骨架對不對、再看血肉、再看行為、最後問靈魂。

**為什麼這個順序**：
- R1 結構錯了 → 後面 R2/R3/R4 都白做（在錯的骨架上談血肉沒意義）
- R2 規則漏了 → R3 策略層的優化會踩坑
- R3 策略對了但 R4 動機錯 → 高效地做錯的事

**例外允許跳層**：
- 簡單議題只需 R1 + R2 → 跑完就停
- 重新審視整體方向時 → 從 R4 開始倒推（先問「為什麼存在」再回頭看結構）
- 使用者明確指定某層 → 直接跑該層

🔴 **任一層發現問題就停下來修，不必跑完 4 層**——這是奧卡姆剃刀的應用，避免把工法本身變成過度工程。

## 路徑 B 完整流程（淬煉一個 md 檔）

```
1. 進場偵察（聽現場主人 + 讀目標 md + 讀相關 references）
2. R1 結構層透視 → 有問題 → 修 → 再透視
3. R2 規則層透視 → 有問題 → 修 → 再透視
4. R3 策略層透視 → 有問題 → 修 → 再透視
5. R4 動機層透視 → 有問題 → 修 → 再透視
6. subagent 三情境衝撞驗證（見 references/三情境衝撞工法.md）
7. 交付給現場主人持久化（直接寫回 md 檔本身）
8. 觸發脈絡回寫（如果動到 SKILL.md 的 description / 觸發條件）
```

**產出形式**：精煉後的 md 檔本身——縮短 + 深化 + 移除殭屍引用 + 修正失效閘門。

## 工法的工法（meta 層）

四層深度透視本身也應該被四層透視——

- **R1 自審**：4 層的順序對嗎？有沒有遺漏的層？（已經審過，4 層是實戰收斂值）
- **R2 自審**：邊緣 case 處理了嗎？（已寫「跳層允許」+ 「停損點」）
- **R3 自審**：4 層會不會變成過度工程？（已寫「任一層停下來」）
- **R4 自審**：誰會用這個工法？淬鍊師被召喚做 md 精煉時 → R4 通過

**結論**：本工法 R1-R4 自審通過，可投入使用。
