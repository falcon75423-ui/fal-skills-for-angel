# 淬煉團召喚指南

> 讀取時機：上游淬鍊時 / 議題影響半徑大時 / 高張力議題時。SKILL.md 「四種武器配置」段指引到這裡。
>
> 核心：**淬煉團是重兵器，不是萬用工具**。

---

## 召喚的判別表

| 情境 | 推薦工具 | 理由 |
|------|---------|------|
| 上游淬鍊（世界觀 / 人設 / 原則 / 跨 Skill 通用智慧）| **淬煉團 5 角色全進場** | 影響半徑大、需要多視角碰撞 |
| 中游淬鍊（閘門 / 條目 / 流程）| subagent 三情境衝撞 | 範圍可控、單一視角夠 |
| 下游淬鍊（偵測機制 / 計數 / 踩雷）| subagent 三情境衝撞 | 機制可量化驗證 |
| 路徑 B（淬煉一個 md 檔）| 4 層深度透視 + subagent 衝撞 | md 精煉是手術，淬煉團是會議——殺雞用牛刀 |
| 高張力設計議題（哪怕是中游）| **淬煉團指定 1-2 個角色** | 多視角破除單視角盲點，但不必全 5 角色 |
| 跨 Skill 議題（影響多個 Skill 的設計） | **淬煉團 5 角色全進場 + 動機追問者站使用者位置** | 跨 Skill = 跨受眾 = 多視角必要 |
| 純 typo / 排版 / 路徑改名 | **不需要任何驗證** | 沒淬鍊內容，直接改 |

🔴 **判別關鍵**：問「這次淬鍊產出影響半徑」+ 「議題張力」兩個維度——影響大 + 張力高 → 淬煉團；其他 → subagent 夠用。

## 召喚的觸發詞

從淬鍊師端召喚淬煉團的字面：

| 召喚層級 | 觸發詞 |
|---------|--------|
| 全 5 角色 | 「淬煉團」「召喚淬煉團」「讓淬煉團來看 X」 |
| 指定單角色 | 「[角色名]，你怎麼看？」「請結構審視者進場」 |
| 指定組合 | 「結構審視者 + 簡化批判者並排看 X」 |

🔴 **形式紀律**：召喚淬煉團 = subagent 真的要進場（透過 Agent 工具）——不能口頭模擬「我來扮演 5 角色」（自指悖論）。

## 5 角色快速參考（完整版見 淬煉團 SKILL.md）

| # | 角色 | 對應層 | 一句話追問 |
|---|------|--------|-----------|
| 1 | 結構審視者 The Architect | 結構層 R1 | 「責任鏈閉環嗎？」 |
| 2 | 邊界挑戰者 The Edge-Case Engineer | 規則層 R2 | 「最痛苦的路徑長怎樣？」 |
| 3 | 策略偵察者 The BS Detector | 策略層 R3 | 「代價呢？症狀還是病因？」 |
| 4 | 動機追問者 The Anthropologist | 動機層 R4 | 「使用者真的需要這個嗎？」 |

詳細人設、底層邏輯、完整話術 → 淬煉團 `references/五角色詳細人設.md`

## 對話節奏（採顧問團規範）

1. **每次發言標角色名稱**：`**[角色名]**：[發言]`
2. **角色之間可互相回應、質疑、補充**——不是 5 個獨立檢查表
3. 🔴 **避免過快共識**——張力產生洞察。5 角色互相認同 = 沒衝撞 = 沒深度
4. 動機追問者常打斷技術討論：「但使用者真的要這個嗎？」
5. 結構審視者建構厚重 + 簡化批判者壓縮輕量化 = **主軸辯證**
6. 最後由淬鍊師（不在 5 角色內）做**定錨整合**

---

## 🔴 淬鍊師的定錨權

5 角色之外，**淬鍊師本身是定錨整合者**。她不在 5 角色內辯論，她在 5 角色之外**整合**。

定錨整合的格式：

```
**淬鍊師定錨**：
- 結構審視者點出 [X]
- 邊界挑戰者證實 [Y]
- 策略偵察者警告 [Z]
- 動機追問者拉回 [A]
- 簡化批判者建議砍 [B]

**下一步**：[整合判斷 + 行動建議]
```

🔴 **定錨權失守的訊號**：
- 淬鍊師被某一角色說服 = 變成 6 角色平行辯論
- 淬鍊師逐字採納 5 角色意見 = 沒做整合
- 淬鍊師沒有「下一步」結論 = 5 角色白跑

## 召喚範例 prompt

呼叫 Agent 工具召喚淬煉團 5 角色時：

```
身份：你是淬煉團 5 角色（結構審視者 / 邊界挑戰者 / 策略偵察者 / 動機追問者 / 簡化批判者）的扮演者。

待淬煉的議題：
{完整描述——背景 + 雛形 + 設計初衷}

請按以下節奏跑：

第一輪：5 角色各自發言（每人 ≤ 100 字）——表態你的初步觀察
第二輪：互相質疑（指名道姓）——「邊界挑戰者，我不同意你說的 X」
第三輪：交鋒收斂——找出爭點 + 各自最強論點
最後：5 角色各自給「最終立場」（一句話，含「採納/否決/有條件採納」字眼）

格式：
**[角色名]**：[發言]

報告 ≤ 1500 字。

提醒：
- 避免過快共識
- 動機追問者要敢打斷技術討論
- 結構審視者建構 + 簡化批判者壓縮 = 主軸辯證
- 不需要「淬鍊師定錨」段——定錨由我（淬鍊師）讀完報告後做
```

## 防止過度使用（盲點）

🔴 **淬煉團是重兵器，重兵器有重兵器的代價**：

| 盲點 | 訊號 | 預防 |
|------|------|------|
| 簡單議題召喚 5 角色 | 5 個角色每個都說「同意 + 補一句」 | 簡單議題用 subagent 三情境就好 |
| 角色脫節 | 5 個角色獨立發言不互相回應 | prompt 強制要求互相質疑 |
| 過快共識 | 5 角色互相點頭 | 強制第二輪「指名質疑」 |
| 定錨權失守 | 淬鍊師被 5 角色帶著走 | 定錨格式硬規則（5 點觀察 + 1 點下一步）|
| 聲音偏向某角色 | 簡化批判者強勢 = 過度做減法 | 5 角色平等發言時間 |
| 淬煉團變決策盤 | 淬鍊師問「該不該做 X」 | 淬煉團是「做得對不對」，「該不該做」找顧問團 |

---

## 跟顧問團的關係

**並存不競爭**：

| 系統 | 性質 | 階段 |
|------|------|------|
| 顧問團 | 戰略決策系統（5 視角碰撞「該不該做」） | 前決策階段 |
| 淬煉團 | 品質審視系統（5 角色碰撞「做得對不對 / 該不該存在」） | 後決策階段 |

**判別**：
- 「我們要不要做 X / 走哪個方向」 → 顧問團
- 「X 已經做完了，這個產出有沒有漏洞」 → 淬煉團

兩者可以接力——顧問團決定方向 → 鍛造主體執行 → 淬煉團驗證品質。

---

## 跟 subagent 三情境衝撞的關係

不衝突——可以接力：

```
低成本快速驗證
  subagent 三情境衝撞（先跑）
    ↓
  發現重大結構性疑慮 / 上游影響大
    ↓
  升級召喚淬煉團
    ↓
  5 角色多輪碰撞 → 淬鍊師定錨
```

🔴 **不要直接跳淬煉團**——除非議題明顯屬上游 / 跨 Skill。先跑 subagent 三情境衝撞，發現需要升級再召喚淬煉團，省成本。

## 工法的工法（meta 層）

淬煉團召喚指南本身——

- **R1 自審**：判別表的維度對嗎？（影響半徑 + 議題張力 = 兩個獨立維度，已涵蓋大多數情境）
- **R2 自審**：邊緣 case 處理？（純 typo 也列了「不需要驗證」邊界）
- **R3 自審**：會不會被當作偷懶藉口？（「都用 subagent 就好不必召喚淬煉團」）→ 風險存在，靠淬鍊師判斷力 + 召喚紀錄事後檢討
- **R4 自審**：淬鍊師會用這個指南嗎？（每次想召喚淬煉團前查一下表 → 是真實使用情境）

**結論**：淬煉團召喚指南 R1-R4 自審通過，可投入使用。
