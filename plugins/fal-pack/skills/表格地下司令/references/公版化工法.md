# 公版化工法

> 讀取時機：使用者選擇公版化時載入。

---

## 靈魂

公版化不是「存一份模板」。是把一份完成的 Excel **結晶成可重複使用的工業標準**——骨架鎖死、接口精確、任何人套用都不會走樣。

---

## 觸發

新建/優化/設計路徑完工後，表哥問：「要將這份表格公版化嗎？」
使用者說「要」→ 進入結晶流程。

---

## 結晶流程

### Phase 1：區域標記

逐 Sheet、逐區域掃描整份 Excel，將每個儲存格標記為三種角色之一：

| 角色 | 定義 | 範例 |
|------|------|------|
| **靜態區** 🔒 | 格式和內容都鎖死，永遠不變 | 表頭標題、Logo 位置、欄位名稱、固定說明文字、邊框結構 |
| **動態區** 📝 | 格式鎖死、內容每次替換 | 客戶名稱、日期、品項、金額、數據欄位 |
| **計算區** 🔢 | 格式和公式都鎖死，值自動計算 | 小計、合計、百分比、公式欄位 |

### Phase 2：逐格確認（🔴 必經——不可省略）

這是防止「每更新一次就發現一個遺漏」的核心機制。

**確認原則**：不是問使用者「這樣 OK 嗎？」然後丟一個 20 項清單。是**逐區域展開**，一次確認一個邏輯區塊。

**確認流程**：

```
1. 表哥先自動標記所有區域的角色（靜態/動態/計算）
2. 按邏輯區塊展示標記結果：
   「Sheet1 標題區（A1:F3）：
    - A1:F1 合併 → 靜態 🔒（主標題 "月度銷售報表"）
    - A2 → 動態 📝（報表月份 — 佔位符 {{報表月份}}）
    - A3:F3 → 靜態 🔒（欄位標題列）
    這樣對嗎？有需要改的嗎？」
3. 使用者確認或修正 → 下一個區塊
4. 重複直到全部區塊確認完畢
```

**🔴 特別注意的確認點**（最容易遺漏的）：

- **合併儲存格的邊框** — 是靜態的一部分，必須確認
- **條件格式規則** — 靜態（規則本身不變）但作用於動態區
- **資料驗證** — 下拉選單的選項是靜態還是動態？
- **公式中的固定參照** — `=B5*1.05` 中的 `1.05` 是靜態常數嗎？以後會改嗎？
- **數值格式碼** — 千分位、百分比、貨幣符號是固定的嗎？
- **隱藏行列** — 有沒有隱藏的參考資料區？是靜態嗎？
- **列印設定** — 列印區域、重複標題列是固定的嗎？
- **凍結窗格位置** — 固定嗎？

### Phase 3：佔位符植入

確認完畢後，在動態區植入佔位符：

**佔位符格式**：`{{中文名稱}}`

**佔位符 Schema**（每個佔位符必須定義）：

```yaml
佔位符:
  - name: "{{客戶名稱}}"
    cell: B2
    type: text          # text / number / date / formula
    required: true
    description: "客戶公司全名"

  - name: "{{報價日期}}"
    cell: B3
    type: date
    format: "yyyy/mm/dd"
    required: true
    default: "today"

  - name: "{{品項名稱}}"
    cell: A10:A25
    type: text
    required: false
    repeat: true         # 可重複（多行填入）
    description: "服務/產品品項"
```

**命名規則**：
- 使用中文名稱，直覺好讀
- 同一概念全模板統一命名（不能 A sheet 叫 `{{總營收}}`、B sheet 叫 `{{revenue_total}}`）
- 名稱不含特殊字元（除了 `{{` `}}` 本身）

### Phase 4：產出物

1. **`.template.xlsx`** — 動態區清空（只保留佔位符文字），格式完整保留
2. **`template_spec.yaml`** — 模板規格檔，包含：
   - 模板版本號（v1.0）
   - 所有佔位符的 schema
   - 靜態區清單（cell 範圍 + 用途）
   - 動態區清單（cell 範圍 + 佔位符 + 型態）
   - 計算區清單（cell 範圍 + 公式）
   - 格式指紋（關鍵 cell 的字型/邊框/填充快照）
3. **存放路徑**：與原始 Excel 同目錄，子資料夾 `templates/`

### Phase 5：驗證

1. 用 `template_spec.yaml` 的佔位符，填入測試資料
2. 產出測試版 Excel
3. 跑完整驗證管線（結構→視覺→相容性）
4. 與原始 Excel 比對格式指紋
5. 全過 → 公版化完成

---

## 換皮流程（使用公版模板）

觸發：使用者說「換皮」並指定模板

```
1. 讀取 template_spec.yaml
2. 開啟 .template.xlsx
3. 詢問使用者提供每個 required 佔位符的值
4. 執行 placeholder 替換（不插行、不動格式）
5. 計算區的公式自動生效
6. 跑驗證管線
7. 交付
```

---

## 版本演化（模板更新）

使用過程中發現需要調整模板：

```
1. 使用者說「這個模板需要改」
2. 開啟 .template.xlsx 進入「優化」路徑
3. 修改完成後重新跑公版化結晶流程
4. template_spec.yaml 版本號 +1（v1.0 → v1.1）
5. 記錄變更點（新增了什麼佔位符、改了什麼靜態區）
```

---

## 重複區域處理（進階）

有些模板有「可重複的行」（如報價單的品項列）：

```yaml
repeat_zones:
  - name: "品項列"
    template_row: 10        # 格式來源行
    range: A10:F25           # 最大填充範圍
    placeholders:
      - "{{品項名稱}}"      # A 欄
      - "{{單價}}"          # B 欄
      - "{{數量}}"          # C 欄
    min_rows: 1
    max_rows: 16
    overflow: "warn"         # 超出範圍時：warn / expand / error
```

處理邏輯：
1. 從 template_row 複製格式到所有填充行
2. 未使用的行保持空白但保留格式和邊框
3. 計算區的公式範圍必須涵蓋 max_rows（如 `=SUM(B10:B25)` 而非 `=SUM(B10:B12)`）
