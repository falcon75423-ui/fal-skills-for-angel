## openpyxl 安全邊界矩陣

### SAFE（可靠操作）

| 操作 | 說明 |
|------|------|
| 新建檔案寫入 cell values | 字串、數字、日期、布林 |
| 新建檔案寫入公式 | 以字串形式寫入，如 `=SUM(A1:A10)` |
| 基本格式設定 | Font, PatternFill, Border, Alignment, NumberFormat |
| 欄寬列高設定 | `column_dimensions[col].width`, `row_dimensions[row].height` |
| 合併儲存格 | `merge_cells()` / `unmerge_cells()` |
| 凍結窗格 | `freeze_panes` |
| 超連結 | `cell.hyperlink` |
| 基本 Auto-filter | `ws.auto_filter.ref` |
| Sheet 操作 | 建立/刪除/重新命名 |
| 新建資料驗證 | 從零建立下拉選單、輸入限制 |
| 新建條件格式 | 從零建立 CellIsRule, FormulaRule, ColorScale |
| 新建圖表 | 從零建立（不是編輯既有） |
| 新建圖片 | 插入新圖片（不是保留既有） |

### UNSAFE（靜默損壞——絕對禁止）

| 操作 | 後果 |
|------|------|
| 開啟+存檔 .xlsm | VBA macro 被剝離，即使用 `keep_vba=True` 也常損壞 |
| Roundtrip 含圖片/形狀的檔案 | 圖片和形狀靜默消失 |
| Roundtrip 含 Sparklines 的檔案 | Sparklines 靜默剝離 |
| Roundtrip 含 ActiveX/Form 控件 | 控件被轉成靜態圖片或消失 |
| Roundtrip 含外部連結的檔案 | 整個 workbook XML 損壞 |
| Roundtrip 含 Power Query/資料連線 | 檔案損壞 |
| Roundtrip 含 OLE 物件 | 靜默消失 |
| Roundtrip 含 Slicers/Timelines | 靜默消失 |
| `data_only=True` 後存檔 | **永久摧毀所有公式**——不可逆 |
| 含註解的 workbook 存檔兩次 | 註解 XML 損壞 |

### RISKY（有時成功有時壞）

| 操作 | 風險 |
|------|------|
| 既有命名範圍 | Workbook scope 通常 OK；Worksheet scope 常壞 |
| 既有條件格式 | 簡單單規則 OK；複雜多規則/公式型常壞 |
| 既有資料驗證 | 版本依賴，有時保留有時靜默丟棄 |
| 既有圖表 | 資料可能保留但格式/版面重設 |
| 既有 Pivot Table | 讀取保留嘗試，但 pivot cache 常壞 |
| 第三方工具產出的檔案 | 非標準 XML 可能無法 roundtrip |

### 根因

openpyxl 把 .xlsx（ZIP 包的 XML）解析成內部物件模型，再序列化回 XML。**不認識的 XML 元素會被靜默丟棄**。所以：
- 新建 = SAFE（只寫自己知道的）
- Roundtrip 簡單檔案 = SAFE
- Roundtrip 複雜檔案 = UNSAFE（不認識的全丟）

## Style 不可變規則

openpyxl 的 Style 物件（Font, PatternFill, Border, Alignment, Protection）是 **immutable by design**。

```python
# ❌ 錯誤 — 直接修改屬性，不會生效或會報錯
cell.font.bold = True

# ✅ 正確 — 建立新物件
from copy import copy
new_font = copy(cell.font)
cell.font = Font(
    name=new_font.name, size=new_font.size,
    bold=True, italic=new_font.italic, color=new_font.color
)
```

複製 cell 格式時必須用 `copy()`：
```python
from copy import copy

def copy_cell_style(source, target):
    target.font = copy(source.font)
    target.border = copy(source.border)
    target.fill = copy(source.fill)
    target.number_format = source.number_format  # str，不需 copy
    target.protection = copy(source.protection)
    target.alignment = copy(source.alignment)
```

## 邊框處理

### 為什麼邊框會消失？

邊框是 **per-cell** 儲存。每個 cell 有 top/bottom/left/right 四條邊。相鄰 cell 的共用邊（如 A1 的 right 和 B1 的 left）是獨立儲存的，後寫入的會覆蓋先寫入的視覺效果。

### 鐵律：邊框最後做一輪獨立 pass

```python
from openpyxl.styles import Border, Side

def set_range_border(ws, min_row, max_row, min_col, max_col, style='thin', color='000000'):
    """資料和格式全部寫完後，最後呼叫這個函數統一設邊框。"""
    side = Side(style=style, color=color)
    for row in range(min_row, max_row + 1):
        for col in range(min_col, max_col + 1):
            cell = ws.cell(row=row, column=col)
            cell.border = Border(
                left=side if col == min_col else side,
                right=side if col == max_col else side,
                top=side if row == min_row else side,
                bottom=side if row == max_row else side,
            )
```

### 合併儲存格邊框

合併後只有左上角 cell 保留格式。但邊框必須對合併區域的四邊界每個 cell 都設定：

```python
def border_merged_range(ws, cell_range, style='thin', color='000000'):
    from openpyxl.utils import range_boundaries
    min_col, min_row, max_col, max_row = range_boundaries(cell_range)
    side = Side(style=style, color=color)
    for row in range(min_row, max_row + 1):
        for col in range(min_col, max_col + 1):
            cell = ws.cell(row=row, column=col)
            cell.border = Border(
                left=side if col == min_col else Side(),
                right=side if col == max_col else Side(),
                top=side if row == min_row else Side(),
                bottom=side if row == max_row else Side(),
            )
```

---

## 合併儲存格注意事項

- 只有左上角 cell 持有 value 和 style
- 寫入非左上角 cell 會靜默失敗或報錯
- Unmerge 後只有左上角 cell 保留值
- 讀取時非左上角 cell 回傳 `MergedCell` 物件，功能受限
- `insert_rows()` 在合併區域中間插入會破壞合併——必須先 unmerge、插入、再 re-merge

## insert_rows 不複製格式

`ws.insert_rows()` 插入空白行，**不會從相鄰行複製任何格式**。這是模板更新時格式消失的第一大原因。

### 安全替代：Placeholder 替換模式

模板中預先放好佔位符，替換時只改 value，格式零損耗：

```python
def fill_template(ws, replacements):
    """替換 {{佔位符}} 而不動格式。"""
    for row in ws.iter_rows():
        for cell in row:
            if cell.value and isinstance(cell.value, str):
                for key, value in replacements.items():
                    placeholder = '{{' + key + '}}'
                    if placeholder in cell.value:
                        if cell.value == placeholder:
                            cell.value = value  # 整格替換，保留型態
                        else:
                            cell.value = cell.value.replace(placeholder, str(value))
```

如果必須插入行，手動從模板行複製格式：
```python
template_row = 4
for i in range(insert_count):
    target_row = 5 + i
    for col in range(1, ws.max_column + 1):
        source = ws.cell(row=template_row, column=col)
        target = ws.cell(row=target_row, column=col)
        copy_cell_style(source, target)
```

## CJK 字寬計算

Excel 的欄寬以「預設字型的 '0' 字元數」為單位。openpyxl 沒有內建 auto-fit。

```python
import unicodedata

def get_display_width(text):
    """計算文字的顯示寬度，CJK 字元算 2 倍。"""
    if not text:
        return 0
    width = 0
    for char in str(text):
        eaw = unicodedata.east_asian_width(char)
        if eaw in ('F', 'W'):  # Fullwidth, Wide（CJK）
            width += 2
        else:  # Narrow, Halfwidth, Neutral, Ambiguous
            width += 1
    return width
```

### 欄寬計算公式

```
width = max(get_display_width(cell_content) for cells in column) + padding
padding = 2~3 字元
```

字型大小調整：`width *= (font_size / 11.0)`
粗體調整：`width *= 1.05`
上下限：`min_width=8, max_width=50`

### 列高計算（wrap_text 時）

```
lines = ceil(content_display_width / column_width)
explicit_lines = text.count('\n') + 1
total_lines = max(lines, explicit_lines)
row_height = max(15, total_lines * font_size * 1.2)  # 單位：點
```

## 模板精讀標準流程

讀取任何既有 Excel 時，必須提取以下所有資訊：

```
Step 1: 結構掃描
  - sheet 名稱清單
  - 每個 sheet 的 used_range
  - merged_cells.ranges

Step 2: 格式提取（逐 cell）
  - font: name, size, bold, italic, underline, color (RGB!)
  - alignment: horizontal, vertical, wrap_text, indent, rotation
  - border: top/bottom/left/right 各自 style + color
  - fill: patternType, fgColor (RGB!)
  - number_format

Step 3: 尺寸提取
  - 每欄 column_dimensions[col].width
  - 每列 row_dimensions[row].height
  - 預設欄寬/列高

Step 4: 頁面設定
  - orientation, paper_size, margins
  - print_area, print_title_rows, print_title_cols
  - header/footer
  - freeze_panes

Step 5: 格式指紋
  - 序列化為 dict，用於產出後自動比對
```

## 數值格式碼參考

| 用途 | 格式碼 | 範例 |
|------|--------|------|
| 千分位整數 | `#,##0` | 1,234 |
| 千分位小數 | `#,##0.00` | 1,234.56 |
| 百分比 | `0.0%` | 12.3% |
| 貨幣（台幣） | `NT$#,##0` | NT$1,234 |
| 貨幣（美元） | `$#,##0.00` | $1,234.56 |
| 日期 | `yyyy/mm/dd` | 2026/03/30 |
| 年份（文字） | `@` 或寫成字串 | 2026（不是 2,026）|
| 零值顯示橫線 | `#,##0;-#,##0;"-"` | - |
| 負數括號 | `#,##0;(#,##0)` | (1,234) |

🔴 年份必須用文字格式或字串，否則 Excel 會加千分位變成 `2,026`。

## 公式安全規則

- 禁止動態陣列函數（FILTER, SORT, UNIQUE, SEQUENCE）— LibreOffice 和 Google Sheets 不完全支援
- 避免 INDIRECT, OFFSET — 揮發性函數，效能殺手
- 優先用 INDEX-MATCH 取代 VLOOKUP — 更穩定
- 公式中禁止使用 Theme Color 的條件格式
- 跨 Sheet 參照用 `SheetName!CellRef` 格式
- 寫完公式後必須在 Python 端獨立計算一次比對結果
