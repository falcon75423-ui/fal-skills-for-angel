# 未完事項追蹤（Pending Followups）

> 看板條目的「未完事項」獨立生命週期管理。
> 進場偵察 step 1 必掃 Active 區——這裡才是真實待辦清單。

---

## 為什麼有這個檔

主案件完工後不再變動（穩定），但「未完事項」會被後續案件滿足（動態）。
兩種穩定度不同的訊息混住 = 寄居者死在房東家。

獨立檔 = 給未完事項獨立的家。可被 reference、可被滿足、自動退役。
進場偵察讀 `_index.md`（看板，主案件靜態歷史）+ 本檔 Active 區（動態真實待辦）。

---

## 寫入紀律

### 案件完工時雙寫

每張案件完工歸檔時：

1. **看板**（`_index.md`）：寫「完工」（**不含未完事項字面**）
2. **本檔**：
   - (a) 滿足舊 PF（如有）-> Active 該筆移到 Satisfied 區（記滿足者 ID + 退役日）
   - (b) 新增本案延伸的 PF（如有）-> 寫入 Active 區

### PF 條目格式

```
- [ ] **PF-{ID}** {一句話描述} | 開單日：{YYYY-MM-DD} | 來源：{案件編號}
```
